Perguntas relevantes sobre GIT e ao GITHUB

  1. O que é GitHub e qual é a sua principal funcionalidade?

    2. É uma plataforma de desenvolvimento colaborativo que aloja projetos na nuvem utilizando o sistema de controle de versões chamado Git. A plataforma ajuda os desenvolvedores a armazenar e administrar o código e faz o registro de mudanças.

  2. Quais são as diferenças entre Git e GitHub?

    3. O Github é uma “rede social dev” em que é possível armazenar e compartilhar projetos de desenvolvimento de software. O Git é um sistema de controle de versão de arquivos

  3. Como criar uma conta no GitHub?

    4. Acesse [github.com](https://github.com), clique em "Sign up", preencha suas informações e siga as instruções.

  4. Quais são os primeiros passos após a criação da conta no GitHub?

    5. Configure seu perfil, adicione uma chave SSH e configure o Git no seu computador.

  5. Como configurar o Git no seu computador local para se conectar ao GitHub?

    6. Instale o Git, configure seu nome e e-mail com `git config --global user.name "Seu Nome"` e `git config --global user.email "seuemail@example.com"`, e adicione sua chave SSH com `ssh-keygen` e `ssh-add`.

  6. O que é um repositório no GitHub?

    7. Um repositório é um espaço onde o código-fonte e o histórico de alterações de um projeto são armazenados.

  7. Como criar um novo repositório no GitHub?

    8. Clique em "New repository" na sua página de perfil, preencha os detalhes e clique em "Create repository".

  8. Qual é a diferença entre um repositório público e um repositório privado?

    9. Repositórios públicos são visíveis para qualquer pessoa, enquanto privados são acessíveis apenas para colaboradores autorizados.

  9. O que significa clonar um repositório e como isso é feito?

    10. Clonar um repositório copia-o para seu computador. Use `git clone `.

  10. Qual é a utilidade de clonar um repositório?

    11. Permite trabalhar localmente com o código-fonte e fazer alterações.

  11. Quais são os principais comandos do Git que são usados com frequência?

    12. - `git init`: Inicia um novo repositório.

    - `git clone`: Clona um repositório existente.

    - `git add`: Adiciona arquivos ao stage.

    - `git commit`: Salva as alterações no repositório.

    - `git push`: Envia alterações para o repositório remoto.

    - `git pull`: Baixa alterações do repositório remoto.

  12. Como fazer commits e qual a importância de uma mensagem de commit clara e descritiva?

    13. Use `git commit -m "Mensagem"` para registrar alterações. Mensagens claras ajudam a entender o histórico do projeto.

  13. O que são branches e como criar uma nova branch no GitHub?

    14. Branches permitem trabalhar em diferentes partes do projeto simultaneamente. Crie uma nova branch com `git branch `.

  14. Como fazer merge de branches e quais são as boas práticas para isso?

    15. Use `git merge ` para unir branches. Boas práticas incluem testar o código e resolver conflitos antes de fazer merge.

  15. Como fazer um fork de um repositório?

    16. Vá para a página do repositório e clique em "Fork" no canto superior direito.

  16. O que é um pull request e como criar um pull request?

    17. É uma solicitação para integrar alterações de uma branch em outra. Crie clicando em "Compare & pull request" após empurrar suas alterações para o repositório.

  17. Como revisar e aceitar um pull request?

    18. Revise na página do pull request e clique em "Merge pull request" para aceitar.

  18. O que são issues e como abrir uma issue no GitHub?

    19. São usados para rastrear bugs e tarefas. Abra uma clicando em "New issue" na aba "Issues".

  19. Como usar o sistema de projetos do GitHub para gerenciar tarefas?

    20. Crie um projeto na aba "Projects", adicione e organize cartões para gerenciar tarefas.
